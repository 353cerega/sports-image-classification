# Sports Image Classification

## Аннотация

В данном проекте мы разрабатываем и обучаем модель компьютерного зрения на основе свёрточных нейронных сетей для классификации изображений по видам спорта. Задача заключается в том, чтобы по фотографии определить, каким спортом занимаются люди на изображении (футбол, баскетбол, теннис и т.д.). Проект охватывает полный цикл MLOps: от сбора и подготовки данных до развёртывания инференс-сервера.

## Постановка задачи

Цель — создать модель, способную распознавать 100 различных видов спорта на изображениях, с последующей упаковкой решения в формате, готовом к продакшену (ONNX, MLflow Serving). Это позволит интегрировать классификатор в веб-приложения, мобильные приложения или системы анализа спортивного контента.

### Зачем это нужно?

- **Образовательные цели**: Проект позволяет изучить и применить современные методы глубокого обучения (PyTorch Lightning), управления экспериментами (MLflow), конфигурации (Hydra) и контейнеризации моделей (ONNX, MLflow Serving).
- **Развлечение и практическая польза**: Такой классификатор может использоваться в спортивных приложениях, например, для автоматического тегирования фото и видео, создания викторин или рекомендаций контента.
- **Потенциал**: Успешное решение может быть в дальнейшем улучшено и адаптировано, например, для распознавания действий в видео.

## Формат входных и выходных данных

Обучающая и тестовая выборки взяты с Kaggle: [Sports Classification Dataset](https://www.kaggle.com/datasets/gpiosenka/sports-classification). Датасет содержит примерно 14 500 изображений в 100 классах, уже разбитых на папки `train`, `valid`, `test`. Каждая подпапка соответствует определённому виду спорта.

- **Входные данные**: изображения в формате `.jpg` произвольного размера (в процессе предобработки приводятся к размеру 224×224 пикселя).
- **Выходные данные**: предсказанный класс (название вида спорта) и вектор вероятностей для всех классов.

## Метрики

Основной метрикой качества является **accuracy** (доля правильно угаданных классов).
В качестве функции потерь используется **кросс-энтропия** (CrossEntropyLoss), стандартная для задач классификации. В процессе обучения логируются значения loss на тренировочной и валидационной выборках (`train_loss`, `val_loss`), что позволяет контролировать сходимость и выявлять переобучение.

## Моделирование

### Подготовка данных

- **Загрузка**: данные скачиваются через Kaggle API (функция `download_data` в `dataset.py`).
- **Аугментация** (только для тренировочного набора):
  - RandomResizedCrop (случайное вырезание области и масштабирование)
  - RandomHorizontalFlip (случайное отражение по горизонтали)
  - Нормализация по каналам со статистикой ImageNet.
- **Предобработка для валидации и теста**:
  - Resize до 256, затем CenterCrop до 224
  - Нормализация (та же, что и для train).
- Все изображения преобразуются в тензоры PyTorch.

### Архитектура модели

Используется предобученная модель [**MobileNetV3Small**](https://docs.pytorch.org/vision/main/models/generated/torchvision.models.mobilenet_v3_small.html) из библиотеки `torchvision`. Последний полносвязный слой заменён на новый с числом выходов, равным количеству классов (100). Все веса, кроме последнего слоя, заморожены для ускорения обучения. Модель обёрнута в `LightningModule` (`SportsClassifier`), что позволяет легко интегрировать её с PyTorch Lightning.

### Обучение

- **Фреймворк**: PyTorch Lightning
- **Управление конфигами**: Hydra (параметры обучения, данных, модели вынесены в YAML-файлы в папке `configs/`)
- **Логирование**: MLflow (метрики, гиперпараметры, чекпоинты)
- **Оптимизатор**: Adam с learning rate = 0.001
- **Количество эпох**: по умолчанию 10 (можно менять через конфиг)

## Установка и настройка

1. **Клонирование репозитория**
   ```bash
   git clone https://github.com/353cerega/sports-image-classification.git
   cd sports-image-classification
   ```
2. **Установка Poetry и зависимостей**
   ```bash
   pip install poetry
   poetry install
   poetry run pre-commit install
   ```
3. **Настройка Kaggle API (для скачивания датасета)**
    Получите токен на Kaggle (Account → Create API Token)
    Сохраните файл kaggle.json в ~/.kaggle/ и установите права:
   ```bash
   chmod 600 ~/.kaggle/kaggle.json
   ```
## Обучение модели

Запуск обучения с параметрами по умолчанию (можно переопределить через командную строку):
```bash
poetry run python -m src.sports_classifier.train
```
Пример переопределения числа эпох и размера батча:
```bash
poetry run python -m src.sports_classifier.train train.max_epochs=5 data.batch_size=32
```
Все параметры описаны в YAML-файлах папки configs/.

## Логирование экспериментов

Метрики автоматически сохраняются в MLflow. Для просмотра запустите сервер UI:
```bash
poetry run mlflow ui
```
Затем откройте в браузере http://127.0.0.1:5000. Примеры графиков можно найти в папке plots/.

## Подготовка модели к продакшену (Конвертация в ONNX)

После обучения можно экспортировать модель в формат ONNX:
```bash
poetry run python -m src.sports_classifier.convert_to_onnx
```
Результат сохраняется в models/model.onnx.

## Инференс-сервер

В проекте реализован сервер на базе MLflow Serving.

Сохраните модель в формате MLflow:
```bash
poetry run python -m src.sports_classifier.save_mlflow_model
```
Запустите сервер:
```bash
poetry run mlflow models serve -m mlflow_model -p 5001 --no-conda
```
Отправка запроса:
```bash
poetry run python -m src.sports_classifier.scripts.predict /путь/к/изображению.jpg
# пример
# poetry run python -m src.sports_classifier.scripts.predict data/test/football/5.jpg
```
